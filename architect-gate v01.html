Exercise: The Architect's Gate

In this activity, you will enforce the Principle of Least Privilege by fixing a broken Role-Based Access Control (RBAC) check.

Scenario: You are auditing a "Delete User" API endpoint. The current code allows anyone with the role "USER" or "MANAGER" to delete accounts. This is dangerous! Only an "ADMIN" should have this power.

Your Task

Modify the delete_account function to restrict access:

  1. Identify the Flaw: Look at the if statement. It currently checks if the role is not "GUEST". This is too permissive.
  2. Fix the Logic: Change the check to explicitly require the role to be "ADMIN".
  3. Handle Rejection: If the user is not an Admin, return "ACCESS_DENIED".
  4. Handle Success: If they are an Admin, return "ACCOUNT_DELETED".

Why This Matters

Least Privilege states that a user should only have the bare minimum permissions necessary to do their job. Permissive checks (like "anyone but Guest") often accidentally grant sensitive powers to lower-level users.
